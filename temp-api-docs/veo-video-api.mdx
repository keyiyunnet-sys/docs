# VEO Video Generation API

Generate high-quality videos using VEO's advanced AI models through an OpenAI-compatible API interface.

## Endpoint Information

<CodeGroup>
```bash Create Video
POST https://api.apimart.ai/veo/videos
```

```bash Query Video Status
GET https://api.apimart.ai/v1/videos/{id}
```
</CodeGroup>

## Authentication

All requests require Bearer token authentication in the header:

```bash
Authorization: Bearer {{your_api_key}}
```

## Create Video Generation

### Request Parameters

<ParamField path="model" type="string">
  Model name. Default: `veo3.1-fast`
  
  Available options:
  - `veo3.1-fast`
</ParamField>

<ParamField path="prompt" type="string" required>
  Video generation prompt describing the desired content.
  
  Example: `"A cat dancing in the rain"`
</ParamField>

<ParamField path="size" type="string">
  Video dimensions. Default: `1280x720`
  
  Options:
  - `1280x720` (landscape)
  - `720x1280` (portrait)
  
  <Note>Use either `size` or `aspect_ratio`, not both</Note>
</ParamField>

<ParamField path="aspect_ratio" type="string">
  Video aspect ratio format:
  - `16:9` (landscape)
  - `9:16` (portrait)
  
  <Note>Use either `aspect_ratio` or `size`, not both</Note>
</ParamField>

<ParamField path="duration" type="integer">
  Video duration in seconds. Supports 8. Default: 8
  
  <Note>Use either `duration` or `seconds`, not both</Note>
</ParamField>

<ParamField path="seconds" type="integer">
  Video duration in seconds. Supports 8. Default: 8
  
  <Note>Use either `seconds` or `duration`, not both</Note>
</ParamField>

<ParamField path="input_reference" type="array">
  Array of reference image URLs to guide video generation. Maximum 3 images.
  
  <Warning>Cannot be used with first/last frame parameters</Warning>
  <Note>Use either `input_reference` or `image_urls`, not both</Note>
</ParamField>

<ParamField path="image_urls" type="array">
  Array of reference image URLs to guide video generation. Maximum 3 images.
  
  <Warning>Cannot be used with first/last frame parameters</Warning>
  <Note>Use either `image_urls` or `input_reference`, not both</Note>
</ParamField>

<ParamField path="first_image" type="string">
  First frame image URL for frame-guided generation.
  
  <Warning>Cannot be used with reference image parameters</Warning>
</ParamField>

<ParamField path="last_image" type="string">
  Last frame image URL for frame-guided generation.
  
  <Warning>Requires `first_image` to be provided. Cannot be used with reference image parameters</Warning>
</ParamField>

### Response Parameters

<ResponseField name="id" type="string">
  Unique identifier for the video generation task
</ResponseField>

<ResponseField name="object" type="string">
  Object type, always `video.generation`
</ResponseField>

<ResponseField name="created" type="integer">
  Task creation timestamp
</ResponseField>

<ResponseField name="model" type="string">
  Model name used for generation
</ResponseField>

<ResponseField name="status" type="string">
  Current status:
  - `queued` - Task is queued
  - `processing` - Video is being generated
  - `completed` - Generation completed
  - `failed` - Generation failed
  - `cancelled` - Task was cancelled
</ResponseField>

<ResponseField name="error" type="string">
  Error message when status is `failed`
</ResponseField>

### Request Example

<CodeGroup>
```bash cURL (Reference Images)
curl --request POST \
  --url https://api.apimart.ai/veo/videos \
  --header 'Authorization: Bearer {{key}}' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "veo3.1-fast",
    "prompt": "A cat dancing in the rain, cinematic style",
    "aspect_ratio": "16:9",
    "seconds": 8,
    "input_reference": [
      "https://example.com/reference1.jpg",
      "https://example.com/reference2.jpg"
    ]
  }'
```

```bash cURL (First/Last Frame)
curl --request POST \
  --url https://api.apimart.ai/veo/videos \
  --header 'Authorization: Bearer {{key}}' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "veo3.1-fast",
    "prompt": "A cat dancing in the rain, cinematic style",
    "aspect_ratio": "16:9",
    "seconds": 8,
    "first_image": "https://example.com/first-frame.jpg",
    "last_image": "https://example.com/last-frame.jpg"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.apimart.ai/veo/videos', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer {{key}}',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'veo3.1-fast',
    prompt: 'A cat dancing in the rain, cinematic style',
    aspect_ratio: '16:9',
    seconds: 8,
    input_reference: [
      'https://example.com/reference1.jpg',
      'https://example.com/reference2.jpg'
    ]
  })
});
```

```python Python
import requests

response = requests.post(
    'https://api.apimart.ai/veo/videos',
    headers={
        'Authorization': 'Bearer {{key}}',
        'Content-Type': 'application/json'
    },
    json={
        'model': 'veo3.1-fast',
        'prompt': 'A cat dancing in the rain, cinematic style',
        'aspect_ratio': '16:9',
        'seconds': 8,
        'input_reference': [
            'https://example.com/reference1.jpg',
            'https://example.com/reference2.jpg'
        ]
    }
)
```
</CodeGroup>

## Query Video Status

Use the returned `id` to check the generation status and retrieve the completed video.

### Request Example

<CodeGroup>
```bash cURL

  --url https://api.apimart.ai/v1/videos/vgen_abc123def456 \
  --header 'Authorization: Bearer {{key}}'
```

```javascript JavaScript
const response = await fetch('https://api.apimart.ai/v1/videos/vgen_abc123def456', {
  headers: {
    'Authorization': 'Bearer {{key}}'
  }
```

```python Python
import requests

response = requests.get(
    'https://api.apimart.ai/v1/videos/vgen_abc123def456',

```
</CodeGroup>

## Image Guidance Modes

VEO supports two mutually exclusive image guidance modes:

<CardGroup cols={2}>
  <Card title="Reference Images Mode" icon="images">
    Use `input_reference` or `image_urls` to provide up to 3 reference images that guide the overall style and content.
  </Card>
  <Card title="First/Last Frame Mode" icon="film">
    Use `first_image` and optionally `last_image` to specify exact start and end frames for the video.
  </Card>
</CardGroup>

<Warning>
You cannot use both modes simultaneously. Choose either reference images OR first/last frame guidance.
</Warning>

## Important Notes

<Warning>
Video generation is asynchronous. Use the returned `id` to query task status and retrieve results.
</Warning>

<Info>
**Parameter Selection Rules:**
- **Size**: Choose either `size` or `aspect_ratio`
- **Duration**: Choose either `duration` or `seconds`  
- **References**: Choose either `input_reference` or `image_urls`
- **Guidance**: Choose either reference images OR first/last frame mode
</Info>

<Tip>
**VEO Features:**
- Fixed 8-second video duration
- Support for both reference image and frame-guided generation
- High-quality output with `veo3.1-fast` model
</Tip>

### Best Practices

1. Use JSON format for all requests
2. Include clear, descriptive prompts for better results
3. For reference mode: Use up to 3 high-quality reference images
4. For frame mode: Ensure first frame is always provided if using last frame
5. Poll the status endpoint every 30-60 seconds until completion
6. Describe visual effects and style preferences in your prompt
