# VEO Create Video Task

Create high-quality video generation tasks using VEO's advanced AI models with support for two image guidance modes.

## Authentication

All requests require a Bearer token in the request header:

```bash
Authorization: Bearer {{key}}
```

## Model Overview

Supported models:

- `veo3_1_fast_stable` (Stable version, 720p only)
- `veo_3_1_fast` (Fast version, supports 720p/1080p/4k, lower quality and prompt control but faster speed, suitable for rapid iteration and quick results)
- `veo_3_1_pro` (High-quality version, supports 720p/1080p/4k, slower generation speed but higher quality and prompt control, suitable for premium quality, professional creation, advertising and experimental short films)

## Request Parameters

<ParamField path="model" type="string">
  Model name, defaults to `veo3_1_fast`
  
  Available options:
  - `veo3_1_fast_stable` - Stable version, 720p only
  - `veo_3_1_fast` - Fast version, 720p/1080p/4k
  - `veo_3_1_pro` - High-quality version, 720p/1080p/4k
</ParamField>

<ParamField path="prompt" type="string" required>
  Video generation prompt describing the desired video content
  
  Example: `"A cat dancing in the rain, cinematic style"`
</ParamField>

<ParamField path="resolution" type="string">
  Output resolution, defaults to `720p`

  Supported values:
  - `720p` - Default, supported by all Veo3.1 models
  - `1080p` - Only supported by veo_3_1_fast, veo_3_1_pro
  - `4K` - Only supported by veo3_1, veo_3_1_fast, veo_3_1_pro

  <Note>Case insensitive: supports 720p/720P/4K/4k and other formats</Note>
  <Warning>veo3_1_fast_stable only supports 720p</Warning>
</ParamField>

<ParamField path="aspect_ratio" type="string">
  Output video aspect ratio (pure ratio), defaults to `16:9`

  Supported values:
  - `16:9` - Landscape
  - `9:16` - Portrait

  <Note>Resolution is specified separately via the resolution field, ratio does not support -1080p/-4K suffixes</Note>
</ParamField>

### Reference Image Mode

<ParamField path="input_reference" type="array">
  Reference image URLs, supports up to three images separated by commas [,]

  Supported models: veo3_1_fast_stable, veo_3_1_fast
  
  <Warning>Cannot be used with first/last frame mode. veo_3_1_pro does not support reference image mode</Warning>
  <Warning>Choose either this or `image_urls`</Warning>
</ParamField>

<ParamField path="image_urls" type="array">
  Reference image URLs, supports up to three images separated by commas [,]

  Supported models: veo3_1_fast_stable, veo_3_1_fast
  
  <Warning>Cannot be used with first/last frame mode. veo_3_1_pro does not support reference image mode</Warning>
  <Warning>Choose either this or `input_reference`</Warning>
</ParamField>

### First/Last Frame Mode

<ParamField path="first_image" type="string">
  First frame image URL

  Supported by all Veo3.1 models

  Usage modes:
  - Used alone: Image-to-video mode
  - Used with last_image: Frame-to-video mode (only some models support last frame)

  <Warning>Mutually exclusive with reference image mode</Warning>
</ParamField>

<ParamField path="last_image" type="string">
  Last frame image URL, must be used with first_image (frame-to-video mode)

  Supported models: veo3_1_fast_stable, veo_3_1_fast, veo_3_1_pro

  <Warning>Must be used with `first_image`, mutually exclusive with reference image mode</Warning>
</ParamField>

## Response Parameters

<ResponseField name="id" type="string">
  Unique identifier for the video generation task, used for subsequent status queries
</ResponseField>

<ResponseField name="object" type="string">
  Object type, always `video.generation`
</ResponseField>

<ResponseField name="created" type="integer">
  Task creation timestamp
</ResponseField>

<ResponseField name="model" type="string">
  Model name used
</ResponseField>

<ResponseField name="status" type="string">
  Task status:
  - `queued` - Queued
  - `processing` - Processing
</ResponseField>

<ResponseField name="error" type="string">
  Error message, returned when status is `failed`
</ResponseField>

## Image Guidance Modes

VEO supports two mutually exclusive image guidance modes:

<CardGroup cols={2}>
  <Card title="Reference Image Mode" icon="images">
    Use `input_reference` or `image_urls` to provide up to 3 reference images for overall style and content guidance
  </Card>
  <Card title="First/Last Frame Mode" icon="film">
    Use `first_image` and optional `last_image` to specify precise start and end frames of the video
  </Card>
</CardGroup>

<Warning>
**Important:** The two modes cannot be used simultaneously, choose only one
</Warning>

## Parameter Selection Rules

<Warning>
**Mutually Exclusive Parameters:**
- **Reference images**: Choose either `input_reference` or `image_urls`
- **Guidance modes**: Reference image mode OR first/last frame mode, cannot use both
- **Model limitations**: veo_3_1_pro does not support reference image mode, veo3_1_fast_stable only supports 720p
</Warning>

<Info>
**Best Practices:**
1. Video generation is asynchronous, use the returned `id` to query task status and results
2. Clearly describe desired visual effects and style in the prompt
3. Choose appropriate model:
   - veo3_1_fast_stable: Stability priority, 720p only
   - veo_3_1_fast: Rapid iteration, supports multiple resolutions
   - veo_3_1_pro: Premium quality, professional creation
4. Choose appropriate guidance mode:
   - Reference image mode: Suitable for style guidance (veo3_1_fast_stable, veo_3_1_fast)
   - First/last frame mode: Suitable for precise start and end control (all models)
5. Select appropriate resolution based on model capabilities
</Info>

<Tip>
**Guidance Mode Selection:**
- **Reference Image Mode**: Use when you want overall style consistency but allow creative variation (only veo3_1_fast_stable, veo_3_1_fast support)
- **First/Last Frame Mode**: Use when you need precise control over video start and end frames (all models support)
- **Resolution Selection**: Choose based on model capabilities and requirements, veo3_1_fast_stable only supports 720p
</Tip>

```java Java (Reference Images)
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.net.URI;
import com.fasterxml.jackson.databind.ObjectMapper;
import java.util.Map;
import java.util.List;

HttpClient client = HttpClient.newHttpClient();
ObjectMapper mapper = new ObjectMapper();

Map<String, Object> requestBody = Map.of(
    "model", "veo_3_1_fast",
    "prompt", "A cat dancing in the rain, cinematic style",
    "aspect_ratio", "16:9",
    "resolution", "1080p",
    "input_reference", List.of(
        "https://example.com/reference1.jpg",
        "https://example.com/reference2.jpg"
    )
);

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.globalaiopc.com/veo/videos"))
    .header("Authorization", "Bearer {{key}}")
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString(mapper.writeValueAsString(requestBody)))
    .build();

HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
```
