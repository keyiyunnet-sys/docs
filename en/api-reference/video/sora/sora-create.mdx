# Sora Create Video Task

Create high-quality video generation tasks using Sora's advanced AI models with support for multiple models and duration options.

## Authentication

All requests require a Bearer token in the request header:

```bash
Authorization: Bearer {{key}}
```

## Request Parameters

<ParamField path="model" type="string" required>
  Model name, available options:
  - `sora2Stable` - Supports 10 or 15 seconds
  - `openAiSora2` - Supports 4, 8, 12 seconds
  - `openAiSora2Stable` - Supports 4, 8, 12 seconds
  - `openAiSora2Plus` - Supports 4, 8, 12 seconds
  - `openAiSora2Pro`  - Supports 4, 8, 12 seconds
</ParamField>

<ParamField path="prompt" type="string" required>
  Video generation prompt describing the desired video content

  Example: `"A cat dancing in the rain, cinematic style"`
</ParamField>

<ParamField path="size" type="string">
  Video dimensions, defaults to `720x1280`

  Available options:
  - `720x1280` - Portrait
  - `1280x720` - Landscape

  <Note>Use either this or `aspect_ratio`, not both</Note>
</ParamField>

<ParamField path="aspect_ratio" type="string">
  Video aspect ratio

  Available options:
  - `16:9` - Landscape
  - `9:16` - Portrait

  <Note>Use either this or `size`, not both</Note>
</ParamField>

<ParamField path="duration" type="integer">
  Video duration in seconds, supports 10 and 15

  <Note>Use either this or `seconds`, not both</Note>
</ParamField>

<ParamField path="seconds" type="integer">
  Video duration in seconds, supports 10 and 15

  <Note>Use either this or `duration`, not both</Note>
</ParamField>

<ParamField path="input_reference" type="array">
  Reference image array to guide video generation, supports up to 1 images

  **Important Notes:**
  - Do not upload real portrait images, as they are likely to fail
  - For `openAiSora2-xxx` series models, image format must match the ratio
  - Only supports 1280×720 (16:9 landscape) and 720×1280 (9:16 portrait)

  <Note>Use either this or `image_urls`, not both</Note>
</ParamField>

<ParamField path="image_urls" type="array">
  Reference image URL array to guide video generation

  **Important Notes:**
  - Do not upload real portrait images, as they are likely to fail
  - For `openAiSora2-xxx` series models, image format must match the ratio
  - Only supports 1280×720 (16:9 landscape) and 720×1280 (9:16 portrait)

  <Note>Use either this or `input_reference`, not both</Note>
</ParamField>

## Response Parameters

<ResponseField name="id" type="string">
  Unique identifier for the video generation task, used for subsequent status queries
</ResponseField>

<ResponseField name="object" type="string">
  Object type, always `video.generation`
</ResponseField>

<ResponseField name="created" type="integer">
  Task creation timestamp
</ResponseField>

<ResponseField name="model" type="string">
  Model name used
</ResponseField>

<ResponseField name="status" type="string">
  Task status:
  - `queued` - Queued
  - `processing` - Processing
</ResponseField>

<ResponseField name="error" type="string">
  Error message, returned when status is `failed`
</ResponseField>

## Parameter Selection Rules

<Warning>
**Mutually Exclusive Parameters:**
- **Size parameters**: Choose either `size` or `aspect_ratio`
- **Duration parameters**: Choose either `duration` or `seconds`
- **Reference images**: Choose either `input_reference` or `image_urls`
</Warning>

## Video Duration Support

| Model | Supported Durations |
|-------|-------------------|
| sora2Stable | 10 seconds, 15 seconds |
| openAiSora2 | 4 seconds, 8 seconds, 12 seconds |
| openAiSora2Stable | 4 seconds, 8 seconds, 12 seconds |
| openAiSora2Plus | 4 seconds, 8 seconds, 12 seconds |

<Info>
**Best Practices:**
1. Video generation is asynchronous, use the returned `id` to query task status and results
2. Clearly describe desired visual effects and style in the prompt
3. Reference images can guide overall style, supports up to 1 images
4. Choose appropriate aspect ratio to match your use case
</Info>

<Tip>
**Prompt Suggestions:**
- Include specific scene descriptions
- Add style keywords (e.g., "cinematic", "artistic", "realistic")
- Describe actions and movement patterns
- Specify lighting and color preferences
</Tip>


```java Java
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.net.URI;
import com.fasterxml.jackson.databind.ObjectMapper;
import java.util.Map;
import java.util.List;

HttpClient client = HttpClient.newHttpClient();
ObjectMapper mapper = new ObjectMapper();

Map<String, Object> requestBody = Map.of(
    "model", "sora-2",
    "prompt", "A cat dancing in the rain, cinematic style",
    "aspect_ratio", "16:9",
    "seconds", 10,
    "input_reference", List.of(
        "https://example.com/reference1.jpg"
    )
);

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://zcbservice.aizfw.cn/kyyReactApiServer/sora/videos"))
    .header("Authorization", "Bearer {{key}}")
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString(mapper.writeValueAsString(requestBody)))
    .build();

HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
```
