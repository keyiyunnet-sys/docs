# Seedance1.5 Query Video Task

Use task ID to query Seedance1.5 video generation task status and results.

## Authentication

All requests require a Bearer token in the request header:

```bash
Authorization: Bearer {{YOUR_API_KEY}}
```

## Path Parameters

<ParamField path="id" type="string" required>
  Unique identifier of the video generation task, returned from create task API
</ParamField>

## Response Parameters

<ResponseField name="id" type="string">
  Unique identifier of the video generation task
</ResponseField>

<ResponseField name="object" type="string">
  Object type, fixed as `video`
</ResponseField>

<ResponseField name="created" type="integer">
  Task creation timestamp
</ResponseField>

<ResponseField name="model" type="string">
  Model name used
</ResponseField>

<ResponseField name="status" type="string">
  Task status:
  - `queued` - Queued
  - `processing` - Processing
  - `completed` - Completed
  - `failed` - Failed
  - `cancelled` - Cancelled
</ResponseField>

<ResponseField name="video_url" type="string">
  Generated video URL, returned when status is `completed`
</ResponseField>

<ResponseField name="actualDuration" type="number">
  Actual video output duration in seconds, returned when status is `completed`
</ResponseField>

<ResponseField name="error" type="string">
  Error message, returned when status is `failed`
</ResponseField>

## Status Flow

<Steps>
  <Step title="queued">
    Task created and added to queue, waiting for processing
  </Step>
  <Step title="processing">
    Task is being processed, Seedance1.5 model is generating video
  </Step>
  <Step title="completed">
    Task completed, video generated successfully, retrieve via `video_url`
  </Step>
</Steps>

## Polling Recommendations

<Info>
**Polling Strategy:**
- Recommended polling interval: 30-60 seconds
- Offline service (flex): 60-120 seconds
- Avoid too frequent requests to prevent rate limiting
- Stop polling when status is `completed`, `failed`, or `cancelled`
</Info>

## Video URL Information

<Warning>
**Important Notes:**
- Task IDs and generated videos are stored for 1 day
- Please download and save videos to your storage system promptly
- Task records and videos will be deleted after 1 day
- Video files are typically in MP4 format
</Warning>

```python Python (Polling Example)
import requests
import time

def wait_for_video(task_id, api_key, max_wait=600, interval=30):
    """
    Poll and wait for video generation completion

    Args:
        task_id: Task ID
        api_key: API key
        max_wait: Maximum wait time (seconds)
        interval: Polling interval (seconds)

    Returns:
        Video URL or None
    """
    url = f'https://zcbservice.aizfw.cn/kyyReactApiServer/v1/result/{task_id}'
    headers = {'Authorization': f'Bearer {api_key}'}

    start_time = time.time()

    while time.time() - start_time < max_wait:
        response = requests.get(url, headers=headers)
        data = response.json()

        status = data.get('status')

        if status == 'completed':
            return data.get('video_url')
        elif status == 'failed':
            print(f"Generation failed: {data.get('error')}")
            return None
        elif status == 'cancelled':
            print("Task cancelled")
            return None

        print(f"Current status: {status}, waiting...")

        time.sleep(interval)

    print("Wait timeout")
    return None

# Usage example
video_url = wait_for_video('video_abc123def456', 'your_api_key')
if video_url:
    print(f"Video generated successfully: {video_url}")
```
